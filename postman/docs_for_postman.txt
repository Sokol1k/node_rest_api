- Login
    Для авторизации необходимо ввести электронную почту пользователя.
    Тело:
        email – электронная почта пользователя.
    В системе всего 3 пользователя:
        1) Продавец-консультант - shop-assistant@gamil.com
        2) Кассир - cashier@gamil.com
        3) Бухгалтер - accountant@gamil.com
    При успешном запросе, тело ответа должно содержать токен.

- Order index
    Возвращает список заказов.
    Обязательно нужно использовать токен типа Bearer.
    Параметры:
        from, to – вывести записи в заданном числовом диапазоне (например: from=2020-02-24 и to=2020-02-25);
        status – вывести все записи с заданным статусом (например: 1 – «в ожидании», 2 – «выполнено», 3 – «оплачено»);
    Доступ имеют: Продавец-консультант, Бухгалтер.
    При успешном запросе, тело ответа должно содержать список заказов.

- Order show
    Возвращает данные о конкретном заказе.
    Обязательно нужно использовать токен типа Bearer.
    Доступ имеют: Продавец-консультант, Бухгалтер.
    При успешном запросе, тело ответа должно содержать данные о заказе.

- Order store
    Создает заказ.
    Обязательно нужно использовать токен типа Bearer.
    Тело:
        product_id – ид продукта, которого заказали.
    Доступ имеют: Кассир.
    При успешном запросе, тело ответа должно содержать данные о заказе, который был только что создан.

- Order update
    Обновляет статус конкретного заказа.
    Обязательно нужно использовать токен типа Bearer.
    Тело:
        status – статус заказа (например: 1 – «в ожидании», 2 – «выполнено», 3 – «оплачено»);
    Доступ имеют: Продавец-консультант, Кассир.
    При успешном запросе, тело ответа должно содержать данные о заказе, который был обновлен.

- Order delete
    Удаляет конкретный заказ.
    Обязательно нужно использовать токен типа Bearer.
    Доступ имеют: Продавец-консультант, Бухгалтер.
    При успешном запросе, тело ответа должно содержать сообщение о успешном удалении.

- Check store
    Создает чек.
    Обязательно нужно использовать токен типа Bearer.
    Тело:
        order_id – ид заказа, для которого создастся чек.
    Доступ имеют: Кассир.
    При успешном запросе, тело ответа должно содержать данные о чеке, который был только что создан.

- Check show
    Возвращает данные о конкретном чеке.
    Обязательно нужно использовать токен типа Bearer.
    Доступ имеют: Кассир.
